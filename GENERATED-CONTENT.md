# Generated Content — OSS Prototype 2

Реестр всего, что добавлено или изменено в прототипе сверх того, что прямо зафиксировано в `docs/oss-prototype/spec/`. Это нужно для проверки и итеративной доработки: Юрий просматривает каждую запись, что-то правит, что-то помечает как `disabled`.

## Что попадает в этот файл

1. **Замены dummy-текстов из фигмы.** В скринах `design/refs/` много dummy- и placeholder-копирайта типа «Несколько слов о том, что …» — это рабочие заглушки, не финальные тексты. При реализации каждая такая заглушка заменяется на нормальный продуктовый текст и логируется здесь.
2. **Новые тексты, которых нет ни в спеке, ни в фигме.** Например, тост `Доступно в продакшене`, валидационные сообщения для конкретных полей, helper'ы под селекторами, popover'ы.
3. **Новые UI-элементы.** Empty-states, loading-skeletons, индикаторы, которых нет ни в спеке, ни в фигме, но без которых интерфейс ломается.
4. **Демо-данные.** Состав квартир в шахматке, ФИО собственников, КН, площади, реквизиты подрядчика, файлы-фейки.

Тексты, которые прямо описаны в спеке (`docs/oss-prototype/spec/*-*.md`, секции «Контент»), сюда НЕ дублируются — спека и так является источником истины.

## Принципы генерации

- **Тексты** — по редполитике Госуслуг (`research/gosuslugi-editorial-policy-official.md`) и Ильяхов:
  - Короткие предложения, без вводных слов.
  - Конкретика > общие слова. Никаких «удобно», «легко», «качественно», «просто».
  - Глаголы действия > отглагольные существительные.
  - Активный залог.
  - Никаких воды, манипуляций, лишней вежливости.
  - Если можно — конкретное число, дата или норма.
- **UX-элементы** — в рамках компонентной базы из `docs/oss-prototype/spec/shared/components.md` и `shared/patterns.md`. Никаких one-off стилей.
- **Domain-факты** — сверяться с `knowledge/oss-prototype/` (особенно `legal-frame.md` и `audit-findings.md`). При необходимости — гуглить (ст. ЖК РФ, письма Минстроя, практика ГЖИ).

## Структура записи

```
### [Короткий ярлык]
- **Локация:** модуль / экран / компонент
- **Тип:** замена dummy / новый текст / новый UI / демо-данные
- **Из фигмы (если замена):** оригинальный dummy-текст
- **Почему:** причина замены / добавления
- **Содержимое:** новый текст или описание элемента
- **Статус:** новое / проверено / disabled
```

`Статус`:
- `новое` — добавлено в этой итерации, ждёт ревью Юрия
- `проверено` — Юрий посмотрел, оставляем как есть
- `disabled` — Юрий решил выключить / убрать из прототипа
